# Create Dotnet Domain Modeling Skill

Created the shared `dotnet-domain-modeling` skill to guide AI agents in modeling .NET domains with Domain-Driven Design, Object-Oriented Design, Clean Architecture, SOLID, YAGNI, rich domain models, aggregate boundaries, and framework-independent business rules.

The skill defines reusable guidance for:

- Modeling entities, value objects, aggregates, domain services, and domain events.
- Protecting invariants through constructors and business methods.
- Avoiding anemic domain models, primitive obsession, public setters, god aggregates, and persistence-driven domain design.
- Keeping domain code independent from frameworks, ORM, infrastructure, API, messaging, and transport concerns.
- Reviewing existing domain code with prioritized findings.
- Creating new domain code with behavior, encapsulation, high cohesion, low coupling, and ubiquitous language.
- Applying practical C# examples for correct and incorrect modeling choices.
- Running a final validation checklist for domain modeling work.
