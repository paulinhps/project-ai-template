# Create Dotnet Clean Architecture Skill

Created the shared `dotnet-clean-architecture` skill to guide AI agents in creating, evolving, reviewing, and maintaining .NET solutions with Clean Architecture, strategic Domain-Driven Design, SOLID, YAGNI, KISS, dependency inversion, bounded context boundaries, and domain independence.

The skill defines reusable guidance for:

- Standard .NET solution and project structure.
- Layer responsibilities for Domain, Application, Infrastructure, and Presentation.
- Allowed and forbidden dependency directions.
- Creation and evolution rules for projects, modules, bounded contexts, and features.
- Architectural review workflow and finding format.
- Prohibited anti-patterns.
- Practical C# examples for correct and incorrect layer usage.
- A final validation checklist for architectural tasks.

